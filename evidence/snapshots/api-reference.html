🔑 baxigpt 开通 API

卡密自助开通 ChatGPT Plus(首月免费试用)。给合伙人 / 自有系统对接用。

Base URL:https://baxigpt.com
鉴权:无需 API Key,卡密(code)本身就是凭据,请勿泄露。
格式:全部 Content-Type: application/json,请求/响应均 JSON。成功 {"ok":true,...},失败 {"ok":false,"msg":"中文原因"}。
限流:/api/submit 每IP 2次/秒;其余 /api/* 8次/秒。另对"卡密错误/垃圾请求"有失败惩罚(刷多了临时封IP),正常使用不受影响。

一、提交开通

POST/api/code-info — 校验卡密余额(可选)

请求:

{ "code": "BX-XXXXXXXX" }

成功:

{ "ok": true, "remaining": 8, "total": 10 }

失败 msg:卡密不存在 / 卡密已停用 / 卡密配额已用完

POST/api/submit — 提交开通(核心)

请求:

{
  "code": "BX-XXXXXXXX",
  "at": "eyJhbGciOi..."
}

at = 账号 access token(eyJ 开头 JWT)。也可整段粘 chatgpt.com/api/auth/session 的 JSON,系统自动提取。长度 ≥30。

⚠️ 仅支持有免费试用资格的新号(没开过 Plus)。

成功(已下单出码,自动付款中):

{
  "ok": true,
  "order_id": "68a850e9...",
  "display_id": "PAY-68A850E9",
  "email": "user@example.com",
  "status": "processing",
  "expires_at": 1780600000,
  "server_now": 1780543000
}

拿到 order_id 后用「查询」轮询到 paid 或 failed/expired。配额规则:成功下单才扣 1 次;失败/退款自动返还,可换号重试;同 code+at 重复提交复用同单不重复扣。

失败 msg(以下情况卡密均自动退回):

msg含义怎么办
该账号已经是 Plus 了,无需开通 ✅本就是 Plus换号
该账号没有免费试用资格…号没资格换新号
access token 已失效,请重新登录…token 失效重取 token
access token 无效或已过期…token 无效重取 token

二、查询

POST/api/status — 查单个订单(order_id 或 at)

{ "order_id": "68a850e9..." }   // 或  { "at": "eyJhbGciOi..." }

响应:

{
  "ok": true, "status": "paid",
  "display_id": "PAY-68A850E9", "order_id": "68a850e9...",
  "email": "user@example.com",
  "expires_at": 1780600000, "server_now": 1780543000
}

status 全集:

status含义卡密
processing开通中(已出码,自动付款,约 1–10 分钟)已扣,待结果
paid✅ 开通成功,可用已消耗
failed出码失败已退回,可重试
expired订单过期未完成已退回,可重试
canceled上游已取消已退回,可重试
轮询建议:提交后每 20–30 秒查一次;processing 继续等,paid 即成功,failed/expired/canceled 即退卡密可重试。

POST/api/query — 按卡密查全部订单+余额

{ "code": "BX-XXXXXXXX" }
{
  "ok": true, "remaining": 7, "total": 10, "used": 3, "status_code": "active",
  "orders": [
    { "display_id": "PAY-68A850E9", "email": "user@example.com",
      "status": "paid", "created_at": "2026-06-04 03:09", "paid_at": "2026-06-04 03:18" }
  ]
}

三、对接示例

# 1) 查余额
curl -s https://baxigpt.com/api/code-info -H 'Content-Type: application/json' -d '{"code":"BX-XXXXXXXX"}'

# 2) 提交开通
curl -s https://baxigpt.com/api/submit -H 'Content-Type: application/json' -d '{"code":"BX-XXXXXXXX","at":"eyJhbGciOi..."}'

# 3) 轮询状态
curl -s https://baxigpt.com/api/status -H 'Content-Type: application/json' -d '{"order_id":"68a850e9..."}'

四、推荐流程

① code-info 确认余额 → ② submit 提交拿 order_id(processing)→ ③ 每 20–30s status 轮询 → ④ paid 收工;failed/expired/canceled 卡密已退、换号重试。

成功开通约 1–10 分钟(取决于上游付款翻牌)。我们不保存任何 access token,仅用于本次开通。
